## What's in the box

A single public function:

```python
from pain001_loader_mt101 import parse_mt101

records = parse_mt101(mt101_text)   # -> list[dict], one per transaction
```

An MT101 (*Request for Transfer*) can carry many transactions (repeating
sequence B), so `parse_mt101` returns one record per transaction. Each
record's keys are exactly the flat-record fields the
[`pain001`](https://github.com/sebastienrousseau/pain001) library
validates against the `pain.001.001.09` JSON schema, so the output feeds
straight into pain.001 generation.

## Why this exists

SWIFT MT-MX coexistence ends in **November 2025**. This loader bridges
the window where upstream systems still emit MT101 but downstream
tooling expects ISO 20022 pain.001.
