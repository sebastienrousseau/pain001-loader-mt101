# pain001-loader-mt101 v0.0.2

Security-maintenance release. Raises the `pain001` floor to propagate a
security fix from the [`pain001`](https://github.com/sebastienrousseau/pain001)
core library. No public API or MT101 → pain.001 mapping changes.

## What changed

- Require `pain001 >= 0.0.55, < 1` (was `>= 0.0.53, < 1`).

The public surface is unchanged — a single function:

```python
from pain001_loader_mt101 import parse_mt101

records = parse_mt101(mt101_text)   # -> list[dict], one per transaction
```

Each record's keys are exactly the flat-record fields the
[`pain001`](https://github.com/sebastienrousseau/pain001) library
validates against the `pain.001.001.09` JSON schema, so the output feeds
straight into pain.001 generation.

## Why this exists

The `pain001` suite releases in lockstep. `pain001 0.0.55` ships a
security fix; this release raises the loader's dependency floor so
installs pull the patched core library.
